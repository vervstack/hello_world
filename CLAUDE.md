## Project Purpose

`hello_world` is a demonstration service for the [Verv stack](https://github.com/godverv), generated by [RedSock CLI (rscli)](https://github.com/Red-Sock/rscli). It is used for testing and demos — not a production service. The repo showcases a key-value store API (Set/Get/Version) backed by SQLite.

## Commands

```bash
# Build
go build ./...

# Run
go run ./cmd/service/main.go

# Test
go test ./...

# Run a single test
go test ./internal/... -run TestName

# Regenerate protobuf + grpc-gateway + swagger from api/grpc/*.proto
make gen   # runs: moti g

# Build Docker image (local, arm64)
make build-local-container
```

## Architecture

The app is structured around three init stages inside `internal/app/`:

1. **Config** (`app.InitConfig`) — loads `config/config.yaml` via `go.vervstack.ru/matreshka`. Config is typed in `internal/config/`.
2. **Data sources** (`app.InitDataSources`) — opens the SQLite connection and runs goose migrations from `./migrations/`.
3. **Servers** (`app.InitServers`) → **Custom** (`Custom.Init`) — starts a single TCP listener using `cmux` to multiplex gRPC (HTTP/2) and HTTP/1.1 on the same port (default `:80`).

### Transport layer (`internal/transport/`)

- `ServersManager` uses `cmux` to route gRPC vs. HTTP traffic on one port.
- `grpcServer` registers protobuf service implementations.
- `httpServer` hosts the grpc-gateway reverse proxy (mounted at `/v1/`) and Swagger UI.
- `docs/` + `pkg/docs/` serve auto-generated Swagger UI via `github.com/Red-Sock/go-swagger-ui`.

### API (`internal/transport/grpc/api/`)

`Impl` is the only gRPC service implementation. It directly queries SQLite — no repository layer. Three RPCs:
- `Version` — returns app version from config.
- `Set` — upserts key/value rows into `user_values`.
- `Get` — reads a single key from `user_values`.

### Proto / code generation

- Proto file: `api/grpc/hello_world_api.proto`
- `moti` manages proto dependencies (`moti.yaml`, `moti.lock`). Dependencies are cached in `proto_modules/` (gitignored).
- Generated Go stubs live in `pkg/hello_world/`.
- `make gen` runs `moti g` which invokes `protoc-gen-go`, `protoc-gen-go-grpc`, `protoc-gen-grpc-gateway`, and `protoc-gen-openapiv2`.

### Files generated by rscli — do not edit

`internal/app/app.go`, `data_sources.go`, `server.go`, `config.go` are regenerated by rscli. Customization goes in `internal/app/custom.go` (marked "DO EDIT, DON'T DELETE").

### Example client

`pkg/client_example/main.go` shows how to connect via gRPC on port 80.

### HTTP example requests

`example/api.http` contains JetBrains HTTP Client requests; set `{{api}}` in `example/http-client.env.json`.
